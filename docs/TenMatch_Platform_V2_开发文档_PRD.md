# TenMatch Platform V2 开发文档（PRD）

> **版本：** V2.0\
> **开发代号：** Platform V2（商业化 + 工作空间）\
> **文档状态：** 第一版

------------------------------------------------------------------------

# 一、项目背景

目前 TenMatch
为单群模式，仅支持官方群运营，所有后台数据均由超级管理员统一管理。

随着合作群计划启动，需要支持：

-   多个合作QQ群接入
-   合作群独立管理后台
-   ToC 用户权益体系
-   爱发电商业化
-   QQ Bot 自动通知
-   平台化架构升级

Platform V2 的目标不是把 TenMatch
做成多个独立群工具，而是构建一个统一的平台。

------------------------------------------------------------------------

# 二、项目目标

Platform V2 包含三大核心模块：

## ToC（用户权益）

-   资料修改解冻卡
-   QQ Bot 站外通知
-   支持者标签

原则：

-   不影响匹配公平
-   不影响评分公平
-   不出售匹配率、曝光率等影响公平性的权益

------------------------------------------------------------------------

## ToB（合作群）

支持合作QQ群接入，每个合作群拥有独立后台。

包括：

-   独立工作空间（Workspace）
-   独立管理员
-   独立运营配置
-   独立统计
-   独立 Bot 配置

------------------------------------------------------------------------

## Platform（平台能力）

统一提供：

-   Workspace（工作空间）
-   权限中心
-   权益中心
-   通知中心
-   CDK 中心
-   Bot 中心
-   系统配置中心

------------------------------------------------------------------------

# 三、核心架构原则（Architecture Principle）

## 1. Workspace 是运营隔离，不是产品隔离

TenMatch 采用 **统一平台、多工作空间** 架构。

Workspace 的作用：

-   合作群运营
-   管理员权限
-   Bot 配置
-   数据统计
-   审核管理

Workspace **不得**作为匹配池隔离单位。

------------------------------------------------------------------------

## 2. 全局统一匹配池（Global Match Pool）

这是 TenMatch 最核心的设计原则。

所有 Workspace 共同组成一个统一匹配池。

``` text
官方群
   │
高校合作群
   │
城市合作群
   │
兴趣合作群
   │
────────────
Global Match Pool
```

所有用户默认进入同一个匹配池。

用户来源群不会影响匹配对象。

例如：

-   官方群用户可以匹配高校群用户。
-   高校群用户可以匹配城市群用户。
-   所有合作群共同建设一个越来越大的生态。

合作群只是运营入口，不是独立产品。

------------------------------------------------------------------------

## 3. 数据分类

### 平台数据（Global Data）

全局共享：

-   用户
-   匹配池
-   匹配关系
-   举报
-   评分
-   权益
-   支持者
-   冷却时间

### 工作空间数据（Workspace Data）

按 Workspace 隔离：

-   Bot
-   管理员
-   公告
-   配置
-   审核日志
-   操作日志
-   运营统计

------------------------------------------------------------------------

# 四、整体架构

``` text
TenMatch Platform
│
├── 官方 Workspace
├── 合作群 A Workspace
├── 合作群 B Workspace
└── ...

           │

Global Match Pool（所有 Workspace 共享）
```

------------------------------------------------------------------------

# 五、Workspace（工作空间）

参考飞书、钉钉、企业微信。

采用：

> 一个账号，可进入多个工作空间。

登录后台：

``` text
TenMatch

▼ 当前工作空间

官方
A合作群
B合作群
```

切换 Workspace 后：

自动切换：

-   用户管理
-   举报
-   审核
-   Bot
-   通知
-   配置
-   统计

------------------------------------------------------------------------

# 六、权限模型

## 超级管理员

拥有：

-   查看全部 Workspace
-   创建 Workspace
-   删除 Workspace
-   修改全部配置
-   查看全部数据
-   切换全部 Workspace

------------------------------------------------------------------------

## Workspace Owner

拥有：

-   本群配置
-   本群统计
-   本群用户
-   本群管理员
-   本群 Bot

不能：

-   查看其它 Workspace
-   修改系统配置

------------------------------------------------------------------------

## Workspace Admin

拥有：

-   用户审核
-   举报处理
-   Bot 通知
-   查看统计

------------------------------------------------------------------------

# 七、合作群管理

支持：

-   新增合作群
-   编辑合作群
-   删除合作群
-   暂停合作
-   延长合作期限

字段：

-   群名称
-   QQ群号
-   Logo
-   简介
-   负责人
-   Bot 状态
-   创建时间
-   到期时间
-   状态

到期：

-   停止 Bot
-   停止新用户加入
-   保留历史数据

------------------------------------------------------------------------

# 八、用户归属

QQ 全局唯一。

一个 QQ：

只能注册一个账号。

支持加入多个 Workspace。

来源 Workspace 仅用于统计分析，不影响匹配逻辑。

------------------------------------------------------------------------

# 九、ToC 权益中心

当前权益：

-   资料修改解冻卡
-   QQ Bot 通知
-   支持者标签

所有权益统一由 UserEntitlement 管理。

------------------------------------------------------------------------

# 十、爱发电 + CDK

采用 CDK 模式。

流程：

``` text
后台生成CDK
↓

导出CSV

↓

导入爱发电

↓

用户投喂

↓

领取CDK

↓

TenMatch兑换

↓

自动发放权益
```

CDK 绑定 Package，不直接绑定单个功能。

------------------------------------------------------------------------

# 十一、通知中心

统一 Notification Center。

通知来源：

-   匹配
-   举报
-   审核
-   系统
-   运营

渠道：

-   站内通知
-   QQ Bot

所有通知统一经过 NotificationService。

------------------------------------------------------------------------

# 十二、Bot 中心

禁止业务直接调用 Bot。

统一由 NotificationService 转发。

Bot 推送失败：

-   不影响站内通知
-   必须写入日志

------------------------------------------------------------------------

# 十三、运营后台

统计：

-   合作群统计
-   CDK 统计
-   权益统计
-   Bot 推送统计

------------------------------------------------------------------------

# 十四、系统配置中心

统一管理：

-   是否开放注册
-   是否开放匹配
-   是否开放照片
-   是否开放评分
-   是否开放 Bot
-   是否开放赞助
-   首页公告
-   维护公告

实时生效。

------------------------------------------------------------------------

# 十六、开发原则

1.  所有后台页面必须根据 Current Workspace 自动过滤运营数据。
2.  匹配池必须全局共享，不允许按 Workspace 隔离。
3.  权限统一由 Role 控制。
4.  通知统一由 NotificationService 分发。
5.  权益统一由 UserEntitlement 管理。
6.  CDK 仅绑定 Package，不直接绑定功能。

------------------------------------------------------------------------

# 十七、最终目标

Platform V2 完成后，TenMatch 将从单群匹配工具升级为具备：

-   全局统一匹配池
-   多工作空间（Workspace）
-   ToC 用户权益体系
-   ToB 合作群运营后台
-   权限中心
-   通知中心
-   Bot 自动通知
-   CDK 商业化
-   系统配置中心

构建一个持续增长、合作群共同运营、统一生态的匹配平台。
