# TenMatch Platform V2 开发文档（PRD）

> **版本：** V2.0\
> **开发代号：** Platform V2（商业化 + 工作空间）\
> **文档状态：** 第一版

------------------------------------------------------------------------

# 一、项目背景

目前 TenMatch
为单群模式，仅支持官方群运营，所有后台数据均由超级管理员统一管理。

随着合作群计划启动，需要支持：

-   多个合作QQ群接入
-   合作群独立管理后台
-   ToC 用户权益体系
-   爱发电商业化
-   QQ Bot 自动通知
-   平台化架构升级

因此，本版本属于平台能力升级，而非普通功能更新。

------------------------------------------------------------------------

# 二、项目目标

Platform V2 包含三大核心模块：

## ToC（用户权益）

当前开放权益：

-   资料修改解冻卡（消耗型）
-   QQ Bot 站外通知（订阅型）
-   支持者标签（身份展示）

原则：

-   不影响匹配公平
-   不影响评分公平
-   不提供任何提高曝光或匹配率的权益

------------------------------------------------------------------------

## ToB（合作群）

支持合作QQ群接入。

每个合作群拥有：

-   独立工作空间（Workspace）
-   独立管理员
-   独立配置
-   独立统计
-   独立Bot配置

官方超级管理员可统一管理全部工作空间。

------------------------------------------------------------------------

## Platform

新增平台能力：

-   Workspace（工作空间）
-   权限中心
-   权益中心
-   通知中心
-   CDK中心
-   Bot中心
-   系统配置中心

------------------------------------------------------------------------

# 三、整体架构

``` text
TenMatch Platform
│
├── 官方 Workspace
├── 合作群 A Workspace
├── 合作群 B Workspace
└── ...
```

所有后台均围绕 Workspace 运作。

------------------------------------------------------------------------

# 四、Workspace（工作空间）

设计参考：

-   飞书
-   企业微信
-   钉钉
-   Slack

采用：

> 一个账号，可进入多个工作空间。

登录后台后：

左上角显示：

``` text
TenMatch

▼ 当前工作空间

官方
A合作群
B合作群
```

切换工作空间后：

以下所有模块自动切换：

-   用户
-   举报
-   审核
-   Bot
-   通知
-   配置
-   统计

------------------------------------------------------------------------

# 五、权限模型

## 超级管理员

拥有：

-   查看全部Workspace
-   创建Workspace
-   删除Workspace
-   修改全部配置
-   查看全部数据
-   切换全部Workspace

------------------------------------------------------------------------

## Workspace Owner（合作群负责人）

拥有：

-   本群配置
-   本群统计
-   本群用户
-   本群Bot
-   本群管理员

不能：

-   查看其它Workspace
-   修改系统配置

------------------------------------------------------------------------

## Workspace Admin

拥有：

-   用户审核
-   举报处理
-   Bot通知
-   查看统计

不能：

-   删除Workspace
-   修改合作群配置

------------------------------------------------------------------------

# 六、合作群管理

新增后台：

## 合作群管理

支持：

-   新增合作群
-   编辑合作群
-   删除合作群
-   停止合作
-   延期合作

合作群字段：

-   群名称
-   QQ群号
-   Logo
-   简介
-   负责人
-   Bot状态
-   创建时间
-   到期时间
-   状态

到期后：

-   停止Bot监听
-   停止新用户注册
-   保留历史数据

每个合作群支持独立功能开关：

-   匹配
-   照片
-   评分
-   举报
-   Bot
-   支持中心

------------------------------------------------------------------------

# 七、用户归属

QQ 全局唯一。

一个 QQ：

只能注册一个 TenMatch 账号。

支持加入多个 Workspace。

例如：

``` text
Harry

官方群
合作群A
合作群B
```

采用 WorkspaceMember 记录归属关系。

------------------------------------------------------------------------

# 八、ToC 权益中心

## 当前权益

### 1、资料修改解冻卡

作用：

解除资料修改冷静期。

后台支持：

-   发放
-   回收
-   使用记录

------------------------------------------------------------------------

### 2、QQ Bot站外通知

同步站内通知。

支持：

-   匹配申请
-   查看申请
-   对方通过
-   冷静期结束
-   照片审核
-   系统公告

用户可自行开启或关闭。

------------------------------------------------------------------------

### 3、支持者标签

投喂成功自动获得：

> ❤ 支持者

展示位置：

-   匹配卡片
-   个人主页
-   查看资料

仅作身份展示。

------------------------------------------------------------------------

# 九、爱发电 + CDK

采用 CDK 模式。

流程：

``` text
后台生成CDK
↓

导出CSV

↓

导入爱发电

↓

用户投喂

↓

获得CDK

↓

TenMatch兑换

↓

自动发放权益
```

无需同步爱发电订单。

------------------------------------------------------------------------

## CDK后台

支持：

-   批量生成
-   导出CSV
-   查询
-   作废
-   撤销

状态：

-   未兑换
-   已兑换
-   已作废

------------------------------------------------------------------------

## Package（权益包）

CDK 绑定 Package。

例如：

### 支持者月卡

包含：

-   QQ Bot通知30天
-   解冻卡×1
-   支持者标签

后续新增权益仅修改 Package 即可。

------------------------------------------------------------------------

# 十、通知中心

统一 Notification Center。

通知来源：

-   匹配
-   举报
-   审核
-   系统
-   运营

通知渠道：

-   站内
-   QQ Bot

流程：

``` text
业务

↓

Notification

↓

站内通知

↓

检查权益

↓

QQ Bot发送

↓

失败写日志
```

------------------------------------------------------------------------

# 十一、Bot中心

所有Bot调用统一经过 NotificationService。

不得业务直接发送消息。

Bot失败：

不得影响站内通知。

必须记录日志。

# 十五、开发原则

## 工作空间隔离

所有后台页面必须依据 Current Workspace 自动过滤数据。

------------------------------------------------------------------------

## 权限统一

统一 Role 权限控制。

------------------------------------------------------------------------

## 通知统一

统一 NotificationService。

------------------------------------------------------------------------

## 权益统一

统一 UserEntitlement。

------------------------------------------------------------------------

## Package 驱动

CDK 不绑定功能。

CDK 仅绑定 Package。

Package 定义：

-   权益
-   数量
-   有效期

-   权限中心
-   通知中心
-   权益中心
-   CDK商业化
-   Bot自动通知
-   系统配置中心
